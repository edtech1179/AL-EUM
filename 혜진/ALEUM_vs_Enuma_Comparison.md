# 에누마(Enuma) vs 알:음(AL:EUM) 비교 분석

> **목적**: 두 서비스의 **영역·대상·기술·가치 제안**을 나란히 놓고 비교해, 알:음이 **무엇을 참고할 수 있고 / 무엇을 의도적으로 다르게 가는가** 를 명확히 한다.
>
> **전제**:
> - 에누마 = **기초 문해·수리 학습** 에듀테크 (Learn-to-Read, Learn-to-Count)
> - 알:음 = **교육정보 접근성** 플랫폼 (Information-Access-to-School)
> - **알:음은 "문맹"이나 "기초학력 결손"을 해결하는 서비스가 아니다.** 알:음이 해결하는 것은 "**한국어로만 제공되는 학교 정보 → 다국어 학부모에게 도달 못함**" 이라는 **정보 전달 문제**다.
> - 따라서 에누마의 "느린 학습자 / 완전 문맹 / 기초학력 보장" 축은 알:음에 직접 이식하지 않는다.
> - 참고하는 것은 **에누마의 UX 철학·제품 설계 원칙·글로벌 확산 전략** 같은 **방법론적 자산**에 한정된다.
>
> **작성**: 혜진팀, 2026-04-17

---

## 1. 한 줄 정의 비교

| | 에누마 (Enuma) | 알:음 (AL:EUM) |
|---|---|---|
| 한 줄 정의 | "세계의 모든 아이가 자기 속도로 **읽고·쓰고·셈하도록** 돕는 앱" | "학교의 **정보**를 다문화 가정의 모국어로 전달하는 AI 서비스" |
| 해결하는 문제 | **학습 능력**의 결손 (문해·수리) | **정보 접근**의 결손 (번역·맥락) |
| 해결 방식 | 게임화된 **학습 콘텐츠** 직접 제공 | 공공 **정보를 수집·변환·전달** |
| 학습자에 요구하는 것 | 앱에서 **매일 학습하는 시간** | **알림을 받고 이해하는 것** (학습 부담 X) |
| 성공의 지표 | 아이가 글을 읽게 됨 | 학부모가 학교에서 무슨 일이 일어나는지 앎 |

> **핵심 구분**: 에누마는 "능력을 길러주는" 서비스, 알:음은 "이미 벌어지고 있는 일을 알려주는" 서비스. **해결 대상 문제 자체가 다르다.**

## 2. 영역(Domain) 비교

```
                  학습·교육 스펙트럼
  ┌──────────────────────────────────────────────────────┐
  │                                                        │
  │  [문해·수리 학습]          [교육 콘텐츠]      [정보 접근] │
  │        ▲                        ▲                ▲      │
  │        │                        │                │      │
  │     에누마                  (EBS, Coursera)     알:음    │
  │   (Todo·글방·                                           │
  │    Kitkit School)                                       │
  │                                                        │
  └──────────────────────────────────────────────────────┘

       ⬅ "능력 구축"                   "정보 전달" ➡
```

- **에누마의 영역**: 학습 콘텐츠 시장 — 수학·영어·한글·읽기의 **커리큘럼을 직접 전달**
- **알:음의 영역**: 학교-가정 정보 소통 시장 — **학교가 이미 발행한 정보**(공지·급식·학사일정)를 **번역·맥락화해 전달**
- 두 영역은 **겹치지 않는다**. 알:음이 다국어 변환하는 것은 NEIS 공지이지 학습 교재가 아니다.

## 3. 대상 사용자 비교

### 3-1. 에누마의 대상

| 층위 | 구체 |
|---|---|
| 일차 학습자 | 학습이 어려운 아이(느린 학습자), 특수교육 대상자, 기초학습 필요 아동 |
| 확장 학습자 | 개발도상국 아동, 난민촌 아동, 일반 아동 |
| 환경 | **교사·교재 부족 환경 포함** — 학습자 혼자서도 작동해야 함 |

### 3-2. 알:음의 대상

| 층위 | 구체 |
|---|---|
| 일차 사용자 | **다문화 가정 학부모** (한국어 미숙, 학교 정보 접근 어려움) |
| 이차 사용자 | 다문화 학생 (특히 중도입국 · 외국인가정) |
| 삼차 사용자 | 담임·다문화 담당 교사, 교육청 |
| 환경 | **학교가 정보를 이미 생산하고 있는 환경** — 알:음은 그 정보를 "번역·맥락화"해 전달 |

### 3-3. 왜 알:음은 "느린 학습자"를 핵심 대상으로 두지 않는가

- 다문화 학부모가 학교 알림을 이해 못 하는 것은 **인지 능력**의 문제가 아니라 **언어**의 문제다.
- 알:음의 설계 기준선은 "**한국어를 거의 못 하는 성인 학부모**"이지 "**학습이 어려운 학생**"이 아니다.
- "쉬운 한국어" 기능을 탑재하더라도 그 목적은 **학부모가 자국어 병기 없이도 필요 시 빠르게 핵심만 파악하도록** 돕는 정보 전달 옵션이지, **학습 능력을 길러주는** 것은 아니다.
- 학생의 기초학력·문해력 문제는 학교·교육청·에누마 같은 **학습 콘텐츠 서비스**의 몫이다. 알:음은 여기에 개입하지 않는다.

## 4. 기술 스택 비교

| 항목 | 에누마 | 알:음 |
|---|---|---|
| 핵심 AI | 적응형 학습 엔진 (문제 난이도 조절) | Gemini 2.5 Flash + RAG (공지 맥락 번역) |
| 데이터 원천 | 자체 제작 커리큘럼·문제은행 | **공공데이터(NEIS) 실시간 API** |
| ML 모델 | 학습자 숙달도 추정 모델 | 학생 유형 분류(RF/XGB) + 학업중단 위험 예측 |
| 콘텐츠 | 수천 개 게임·문제·도서 | 교육 용어 KB (250개 × 6개 언어) |
| 딜리버리 | 전용 앱 (iOS/Android 태블릿) | PWA + 카카오톡/문자 + 교사 대시보드 |
| 오프라인 | 지원 (핵심 기능) | 불필요 (알림 기반) |
| 학습자 센서 | 터치 입력, 학습 로그 | 학부모 열람 로그, 자녀 학교 정보 |

## 5. 수익·확산 구조 비교

| | 에누마 | 알:음 |
|---|---|---|
| B2C | 토도 시리즈 앱 구독 | (없음) |
| B2B | 토도 클래스룸 (미국 ~7,000교실) | 교육청 · 학교 대시보드 (계획) |
| B2G | 한국 취약계층 4,000명 (공공 예산 연계) | 교육부 · 시도교육청 연계 (핵심 타깃) |
| ODA | 인도네시아·말레이시아·이라크·탄자니아 | (현재 계획 없음 — 국내 확산 우선) |
| 수상 | Global Learning XPRIZE ($15M), Parents' Choice Gold, Google Play Best | (공모전 단계) |
| 다운로드 | 전 세계 1,300만+ | (MVP 단계) |

**관찰**: 에누마는 **B2C 구독 → B2B 학교 → B2G ODA** 순으로 확장. 알:음은 **B2G(교육부) 중심에서 출발**하여 **공공데이터 기반의 공공 서비스로 포지셔닝** 하는 것이 적합. 에누마 모델을 그대로 따를 필요는 없다.

## 6. 가치 제안 비교

### 6-1. 공통점 (둘 다 추구)

- **교육 형평성 (Educational Equity)** — 출신 배경과 무관하게 동등한 교육 기회
- **AI 기반 맞춤화** — 학습자/사용자 특성에 따른 차별적 대응
- **접근성 우선 설계** — 기술 리터러시가 낮은 사용자도 쓸 수 있어야 함

### 6-2. 차이점

| 관점 | 에누마 | 알:음 |
|---|---|---|
| 교육 형평성의 정의 | **학습 능력을 기를 기회의 평등** | **학교 정보를 받을 기회의 평등** |
| AI 맞춤화의 목적 | 학습자 숙달도에 따라 **다음 문제 난이도 조절** | 학부모 언어·유형에 따라 **전달 방식 조절** |
| 접근성의 구현 | UDL (이미지·음성·자판 등 다양한 입력) | 다국어 UI + 큰 글씨 + 음성 재생 + 카카오톡 알림 |

## 7. 알:음이 **참고할 수 있는** 에누마 자산 (선별 수용)

에누마에서 **알:음 영역에 맞게 골라서 쓸 수 있는** 것들. "문해·느린 학습자" 축은 제외하고, **방법론·UX·확산 전략** 관점에서만 차용한다.

| 참고 요소 | 알:음 적용 방식 | 수용 여부 |
|---|---|---|
| **게이미피케이션 UX** (배지·스트릭·진도) | 학부모 알림 연속 확인 배지, 주간 학교 정보 탐험 진도 | ✅ 선별 수용 |
| **다양성 반영 일러스트** (피부색·외형·가족 형태) | 알:음 UI 아이콘·이미지에 문화 중립성 적용 | ✅ 수용 |
| **적응형 AI 개념** (학습자 수준에 맞춤) | 학부모 한국어 수준별 번역 복잡도 자동 조절 (Krashen i+1) | ✅ 수용 |
| **UDL 원칙** (모두가 쓸 수 있는 UI) | 큰 글씨 · 음성 재생 · 아이콘 병기 | ✅ 수용 |
| **다국어 확장 아키텍처** | 6개 언어 동등 품질 테스트 + 원어민 검수 | ✅ 수용 |
| **ODA 수출 모델** | 현 단계에서는 **검토만**, 국내 확산 우선 | ⚠️ 보류 |
| **자체 커리큘럼 설계** | 알:음은 커리큘럼을 만들지 않음 | ❌ 수용하지 않음 |
| **문해·수리 학습 기능** | 알:음 범위 밖 | ❌ 수용하지 않음 |
| **"느린 학습자" 카테고리** | 알:음 범위 밖 | ❌ 수용하지 않음 |
| **기초학력 보장 기능** | 알:음 범위 밖 (학교·교육청·에누마의 역할) | ❌ 수용하지 않음 |
| **자기주도 "학습"** | 알:음은 학습이 아닌 "정보 이해"를 돕는 서비스 | ❌ 수용하지 않음 |

## 8. 알:음이 **의도적으로 다르게 가는** 5가지

1. **학습 콘텐츠를 만들지 않는다.** 알:음은 학교가 이미 발행한 정보(NEIS)를 변환해 전달할 뿐.
2. **기초학력·문해력에 개입하지 않는다.** 다문화 학생의 학습 부진 원인이 "언어 장벽"이든 "학습 속도"든, 알:음은 이를 진단·교정하지 않는다.
3. **"자기 속도의 학습"이라는 에누마 슬로건을 사용하지 않는다.** 알:음은 학습 서비스가 아니기 때문에, 사용자에게 "학습 부담"을 주지 않는 것이 원칙.
4. **학부모를 "학습자"로 위치시키지 않는다.** 학부모는 "정보를 받는 주체"다. 물론 한국어가 자연스럽게 늘 수는 있지만, 그것이 서비스 목적은 아니다.
5. **"느린 학습자" 시장을 타깃으로 하지 않는다.** 다문화 학생 중 학습 지원이 필요한 경우는 **학교 내 기초학력 보장 프로그램**(에누마·교육청 프로그램 등)으로 연결(referral)해야 할 영역.

## 9. 협력 가능성 — "경쟁"이 아닌 "상호보완"

두 서비스가 겹치지 않는다는 것은 **경쟁 관계가 아니라 파이프라인 관계** 임을 의미한다.

```
[학교가 발행]          [가정이 이해]         [학생이 학습]
   NEIS 공지     →     알:음 다국어 변환  →   에누마 토도한글
                      (정보 접근성)          (학습 콘텐츠)
                        ↑                      ↑
                  "학부모가 알도록"      "학생 스스로 기르도록"
```

**구체 협력 시나리오**:
- 알:음이 "우리 학교는 기초학력 보장 프로그램을 운영합니다" 공지를 번역해 학부모에게 전달 → 학부모가 **에누마 토도한글 신청**
- 에누마 토도 클래스룸 도입 학교에서 알:음을 학부모 소통 채널로 동시 활용
- 단, **공식 MOU는 현 단계에서 불필요**. 각자 독립적으로 성장하면서 자연스러운 연동 지점만 확보.

## 10. 결론 — 두 문장 요약

1. **알:음은 에누마가 아니다.** 학습 콘텐츠 서비스가 아닌, 공공 교육정보 접근성 AI 플랫폼이며, 문맹·느린 학습자·기초학력 결손은 알:음이 해결하는 문제가 아니다.
2. **그러나 알:음은 에누마의 방법론적 자산에서 배운다.** 게이미피케이션 UX, 다양성 내장 디자인, 적응형 난이도, UDL 접근성 — 이 네 가지는 **학습 서비스 영역을 건드리지 않으면서도** 알:음의 정보 전달 품질을 높이는 데 유용하다.

## 부록 B — FAQ

**Q1. 알:음에 "쉬운 한국어" 기능을 넣는 건 결국 문해·학습 문제에 개입하는 것 아닌가?**
A. 아니다. 쉬운 한국어는 **정보 전달의 가독성 옵션**이지, 학부모의 한국어 **능력을 기르기 위한 훈련**이 아니다. 목적·측정 지표·사용자 경험이 모두 다르다.

**Q2. 에누마와 파트너십 맺으면 더 강하지 않을까?**
A. 현 단계에서는 **불필요**. 각자 독립적으로 성장하는 것이 더 건강하다. 장래에 MEGI 상위 지역 파일럿에서 "알:음 알림 → 에누마 토도한글 안내" 같은 자연스러운 연계가 나타나면 그때 논의해도 늦지 않다.

**Q3. 그래도 "느린 학습자 지원" 기능이 있으면 좋은 것 아닌가?**
A. 알:음의 타깃은 **학부모**다. 학부모에게 "당신 자녀는 느린 학습자입니다"라고 말하는 서비스가 될 위험이 있다. 학생 개인의 학습 속도 문제는 **학교 내 기초학력 전담교사·교육청·에누마 같은 학습 콘텐츠 서비스**의 몫으로 남겨두는 것이 올바르다.

**Q4. 에누마 강연(김현주, 2026.04.11)에서 "느린 학습자를 위한 책 제작"을 강조했던데?**
A. 그것은 **에누마의 미션**이다. 알:음이 같은 미션을 가져야 할 이유는 없다. 알:음은 "다문화 가정의 학교 정보 접근성"이라는 **독립적인 미션**을 가진다. 에누마 강연에서 배울 것은 **"어떻게 포용적으로 설계하는가"의 방법론**이지 "느린 학습자를 대상으로 삼아라"는 **대상 범위**가 아니다.
