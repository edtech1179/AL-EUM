## 0. 냉정한 진단 — 현재 기획의 3가지 약점

### 약점 1. "맥락"이라는 단어가 **번역 + α** 수준에서 막혀 있다

현재 기획서가 "맥락"이라고 부르는 것을 해부하면:

| 기획서 상 "맥락" | 실제 정체 | 심사위원의 반박 가능성 |
|---|---|---|
| 교육 용어 KB 번역 (수행평가 → "성적에 반영되는 과제") | **번역 사전** | "Google 번역에 LLM 프롬프트로 '쉽게 풀어써줘' 붙이면 되는 것 아닌가?" |
| Action Item 추출 | **요약 + 불릿화** | "Gemini에 '할 일만 뽑아줘'라고 프롬프트 넣으면 되는 것 아닌가?" |
| 마감일 D-N 표시 | **날짜 계산** | "이게 AI가 필요한가?" |
| 학교 약어 해석 (스클 → 스포츠클럽) | **사전 + 치환** | "사전 매칭이면 AI 아닌 정규식 아닌가?" |

**심사 1순위 "활용가능성"과 2순위 "AI 활용도"를 동시에 방어하려면, 위 4개만으로는 부족하다.** 핵심은 — **"번역기로는 절대 도달할 수 없는 정보 재구성"** 을 보여야 한다는 것.

### 약점 2. "AI 없으면 작동 안 한다" 주장이 **사후적 합리화**처럼 보인다

기존 p.6 "AI 의존도 분석"의 주장:
> "AI 제거 시 6개 핵심 기능 전부 작동 불가"

하지만 심사위원이 뜯어보면:
- 번역 → Google Translate로 대체 가능
- Action Item 추출 → 규칙 기반 NLP로 대체 가능
- 유형 분류 → 3개 질문 룰엔진으로 대체 가능

→ **"AI 활용도" 만점 논거가 허술하다.**

### 약점 3. 심사 "독창성" 3순위에서 **핸드트래킹이 역효과**를 낼 가능성

- 핸드트래킹은 "멋있어 보이지만" **학부모 대상 학교 정보 서비스**와 톤이 안 맞음
- 심사위원: "학부모가 웹캠 앞에서 손을 흔들며 학교 알림을 읽는다고요?"
- 이건 제거하거나, **학생 전용 부가 모듈**로 격하 필요

## 1. "맥락"의 재정의 — 10개 축으로 세분화

**맥락(Context) = 같은 정보가 누구에게 · 언제 · 어떤 상황에서 주어지느냐에 따라 달라지는 해석의 층**

번역기는 **언어 층** 하나만 바꾼다. 알:음은 아래 10개 층을 **동시에 재구성**한다.

| # | 맥락 축 | 정의 | 번역기로 가능? | 알:음 AI 구현 |
|---|---|---|---|---|
| 1 | **언어** (Linguistic) | 한국어 → 모국어 | ✅ | RAG + 용어 KB로 **정확도 보강**만 |
| 2 | **제도** (Institutional) | 한국 교육 제도 상 의미 | ❌ | "수행평가 → 내신 반영 → 입시 연결" 제도 체인 KB |
| 3 | **시간** (Temporal) | 학사일정 흐름 상 위치 | ❌ | NEIS 학사일정 타임라인 그래프 + "지금 이 공지가 온 이유" |
| 4 | **누적** (Cumulative) | 과거 공지와의 연결 | ❌ | 이전 공지 임베딩 벡터 검색 + "이 공지는 3주 전 예비안내의 후속" |
| 5 | **학생** (Student-specific) | 이 학생에게 특별히 무엇을 의미하는가 | ❌ | 학년·반·유형·한국어 수준 기반 개인화 재작성 |
| 6 | **가정** (Family) | 이 가정에 실제로 뭐가 문제인가 | ❌ | 종교·거주·형제 수 기반 리스크 플래그 |
| 7 | **학교** (School-specific) | 이 학교 특유의 관행·프로그램 | ❌ | 학교알리미 + NEIS 해당 학교 데이터 교차 |
| 8 | **문화** (Cultural) | 모국 교육 문화와의 차이 | ❌ | 출신국별 "한국 학교는 다릅니다" 비교 코멘트 |
| 9 | **행동** (Action) | 우선순위·리스크 있는 To-Do | △ | Action Item + **리스크 가중치** + **우선순위 정렬** |
| 10 | **이력** (History) | 이 학부모의 과거 열람 패턴 | ❌ | 학부모 행동 로그 기반 "놓치기 쉬운 포인트" 강조 |

> **심사위원에게 보여줄 한 장짜리 비교**: Google 번역은 축 1만, 알:음은 축 1~10 **동시 재구성**. 이게 "번역 ≠ 맥락"의 정량적 증명.

## 2. 10개 맥락 축별 구체 구현 — 기존 기획서 어디에 무엇을 넣을 것인가

### 축 1. 언어 맥락 (Linguistic Context) — *기존 p.7 유지 + 정량 강화*

**현재 위치**: 기존 p.7 "교육 맥락 지능형 변환 엔진" 이미 존재
**보강**: Google 번역과의 A/B 테스트를 **10건 → 50건으로 확대** + **교육부 검수 위원** 3명 평가 포함

### 축 2. 제도 맥락 (Institutional Context) — 🆕 신규 섹션

**구현**: **한국 교육 제도 체인 KB** 구축
- 수행평가 → 내신 → 대학 입시 연결 그래프
- 생활기록부 → 추천서·입시 자료 연결
- 방과후학교 → 보충학습 → 학력 격차 연결
- 학업중단 → 검정고시 → 재입학 경로 연결

**AI 활용**:
- Gemini가 **공지에 등장한 교육 용어** 감지 → 제도 체인 KB에서 관련 노드 검색 → "이 공지가 **왜 중요한지**"를 입시/진로 관점에서 2문장으로 보강

**예시 출력 (베트남어)**:
```
원문: "중간고사 일정 안내 — 5월 20일(월) 1~3교시"

알:음 출력:
📅 중간고사 (Kỳ thi giữa kỳ) — 5/20 월요일

💡 왜 중요한가요?
   한국에서 중간고사 성적은 내신(생활기록부)에 기록되며,
   고등학교의 경우 대학 입시에 직접 반영됩니다.
   (베트남의 일반 정기평가와 다릅니다 — 누적됩니다)
```

**→ Google 번역은 절대 이걸 못 한다.** 제도 지식 없이는 불가능.

### 축 3. 시간 맥락 (Temporal Context) — 🆕 신규

**구현**: NEIS 학사일정 전체를 **연간 타임라인**으로 구조화
- 3월(신학기 적응) → 5월(체험학습 집중) → 7월(1학기 종료) → 9월(2학기 시작) → 12월(진로 상담) → 2월(학년말)
- 각 공지가 **이 타임라인의 어느 구간**에 속하는지 자동 태깅

**AI 활용**:
- 공지 입력 → 시간 맥락 분류기 (Gemini few-shot) → "이 시기의 전형적 이슈" 자동 첨부

**예시 출력**:
```
📅 5월: 체험학습 집중 시기
   이번 공지는 5월에 몰리는 체험학습 3~4건 중 하나입니다.
   앞으로 2~3건 더 올 수 있어요. 모든 동의서·도시락 준비 체크하세요.
```

### 축 4. 누적 맥락 (Cumulative Context) — 🆕 신규 ⭐ 독창성 핵심

**구현**: **공지 간 연결성 추적 엔진**
- 모든 과거 공지를 임베딩 벡터화 → Firestore 저장
- 새 공지 수신 시 → **의미적으로 관련된 과거 공지** 3건 자동 검색 → "**이 공지는 ○○의 후속입니다**" 헤더 자동 생성

**AI 활용**:
- Vertex AI text-embedding-005로 임베딩
- 코사인 유사도 0.75+ 관련 공지 후보
- Gemini가 **연결 서사** 생성: "3월 15일 체험학습 예비안내 → 4월 20일 장소 확정 → **5월 1일 동의서 제출**"

**예시 출력**:
```
📢 오늘의 공지 (5/1): 체험학습 동의서 제출

🔗 이 공지는 이전 공지의 후속입니다:
   ① 3/15 체험학습 예비안내 (읽음 ✅)
   ② 4/20 장소 확정 안내 (놓침 ⚠️)
   ③ 5/1 동의서 제출 (지금 여기) ← 마감 D-3

⚠️ 4/20 공지를 놓치셨기 때문에 장소(남산 서울타워)를 
   놓치셨을 수 있습니다. 동의서 작성 전 확인하세요.
```

> **이것이 핵심 독창성**: 단일 공지를 번역하는 게 아니라, **학부모의 정보 이력 전체를 관리**하는 AI.

### 축 5. 학생 맥락 (Student-specific Context) — *기존 p.8 확장*

**현재 위치**: 기존 p.8 "학생 유형 진단" 이미 존재
**보강**: 4가지 추가 차원
- 학년별 맥락 (초1 vs 고3이 같은 공지를 달리 해석)
- 과거 참여 이력 ("지난 체험학습에 참여했음")
- 형제·자매 재학 여부 (동시 공지 병합)
- 학생 유형별 행동 경로

**AI 활용**:
- Gemini가 학생 프로필 벡터 + 공지 벡터 → 개인화 재작성
- 예: "고3 자녀의 중간고사 = 수시 대비 마지막 시험" vs "초3 자녀의 중간고사 = 연습 수준"

### 축 6. 가정 맥락 (Family Context) — 🆕 신규

**구현**: 가정 프로필 4요소
- 종교 (할랄 여부)
- 거주 지역 (통학 거리)
- 형제 수 (공지 병합)
- 과거 참여도 (낮으면 Action Item 강조)

**AI 활용**:
- 급식 공지 수신 → 가정 종교 체크 → 돼지고기/알코올 포함 시 🔴 경고 자동 삽입
- 체험학습 공지 → 거주지 기준 집결지 안내 생성

**예시 출력** (무슬림 가정):
```
🍽️ 5/7 급식: 돈까스, 미소된장국

🔴 할랄 경고
   돈까스는 돼지고기입니다. 자녀에게 도시락을 준비해주세요.
   (학교에 대체 급식 신청도 가능 — [신청서 양식])
```

> Google 번역은 "Pork" 번역만 해준다. 알:음은 **가정 상황 기반 액션까지 제시**.

### 축 7. 학교 맥락 (School-specific Context) — 🆕 신규

**구현**: 학교알리미 API + NEIS 학교기본정보 결합
- 이 학교의 다문화 학생 비율
- 이 학교의 한국어학급 유무
- 이 학교의 방과후 프로그램 종류
- 이 학교 다문화 담당교사 정보

**AI 활용**:
- 공지를 **이 학교 맥락**으로 해석: "우리 학교는 방과후 한국어반이 있습니다 → 자녀 등록 고려"

### 축 8. 문화 맥락 (Cultural Context) — 🆕 신규 ⭐ 독창성 2

**구현**: 출신국별 **교육 문화 비교 KB** (6개국 × 50개 토픽)
- 한국: 도시락 문화 있음 vs 베트남: 학교 급식 기본
- 한국: 학부모 참여 적극적 vs 중국: 교사 전권
- 한국: 학생회·동아리 활발 vs 필리핀: 종교 활동 중심

**AI 활용**:
- 공지의 토픽 자동 분류 → 출신국 문화와의 차이점이 있으면 자동 코멘트

**예시 출력** (베트남 가정 · 체험학습 공지):
```
✅ 도시락 준비해주세요

💡 베트남과의 차이
   베트남 학교는 체험학습 시 학교에서 도시락을 제공하지만,
   한국 학교는 가정에서 준비합니다. 학교가 잊은 게 아닙니다.
```

### 축 9. 행동 맥락 (Action Context) — *기존 p.7 확장*

**현재 위치**: Action Item 추출 이미 존재
**보강**:
- **우선순위**: 마감 긴급도 + 학부모 영향도 곱연산
- **리스크**: 놓쳤을 때 손실(체험학습 참여 불가 등)을 명시
- **의존성**: "A를 해야 B가 가능"의 순서 표시

**AI 활용**:
- LLM이 각 Action에 **priority_score** (0~100) + **risk_tag** (낮/중/높) + **depends_on** (선행 Action) 부여

**예시 출력**:
```
✅ Action Items (우선순위 정렬)

🔴 긴급 [우선순위 95, 리스크 높음]
   1. 동의서 서명 (D-3)
      → 미제출 시 체험학습 참여 불가

🟡 보통 [우선순위 60]
   2. 도시락 준비 (5/15 당일)
      → [선행조건: Action 1 완료]

🟢 선택 [우선순위 30]
   3. 학부모 동행 여부 응답
```

### 축 10. 이력 맥락 (History Context) — 🆕 신규

**구현**: 학부모 행동 로그 분석
- 평균 열람 소요 시간
- 놓친 공지 패턴 (시간대·주제)
- 질문 챗봇 이력 (자주 혼란스러워하는 주제)

**AI 활용**:
- 이 학부모가 "체험학습"을 과거 2번 놓침 → 새 체험학습 공지에 자동 강조 배지
- 이 학부모가 "수행평가"를 챗봇에 3번 질문 → 수행평가 공지 시 자동 확장 설명

## 3. "그냥 번역"과의 정량 비교 테이블 (심사 슬라이드 1장용)

| 평가 축 | Google 번역 | Papago | ChatGPT 프롬프트 | **알:음** |
|---|---|---|---|---|
| 언어 번역 정확도 | 직역만 | 직역만 | 맥락 일부 | **교육 용어 RAG 보강** |
| 제도 맥락 | ❌ | ❌ | △ (제한적) | ✅ 제도 체인 KB |
| 시간 맥락 | ❌ | ❌ | ❌ | ✅ NEIS 학사일정 연동 |
| **누적 맥락** | ❌ | ❌ | ❌ | ✅ **이전 공지 자동 연결** |
| 학생 맥락 | ❌ | ❌ | ❌ (공지 단독 처리) | ✅ 유형 × 학년 × 이력 |
| 가정 맥락 | ❌ | ❌ | ❌ | ✅ 종교·거주·형제 반영 |
| 학교 맥락 | ❌ | ❌ | ❌ | ✅ 학교알리미 + NEIS 연동 |
| 문화 맥락 | ❌ | ❌ | △ | ✅ 6개국 × 50토픽 비교 KB |
| 행동 맥락 | ❌ | ❌ | △ (단순 불릿) | ✅ 우선순위 + 리스크 + 의존성 |
| 이력 맥락 | ❌ | ❌ | ❌ | ✅ 학부모 행동 로그 |

→ **10개 축 중 Google/Papago는 1개, ChatGPT는 3개, 알:음은 10개.**

> **이것이 "번역이 아닌 맥락 재구성 AI" 주장의 정량적 증명.**

## 4. "AI 없이는 작동 불가"를 진짜로 증명하는 구조

### 멀티 에이전트 아키텍처 — 기존 p.6 교체 ⭐

기존 기획서의 "5개 모듈"을 **협업하는 AI 에이전트 파이프라인**으로 재설계:

    │
    ▼
┌─────────────────────────────────────────┐

**이 구조의 AI 활용도 주장**:
- **6개 에이전트 중 4개가 LLM 추론 기반**
- **병렬 검색**은 RAG 기술
- **팩트 체킹 레이어**는 LLM-as-Judge 패턴
- **에이전트 간 협업**은 2024~2026년 AI 트렌드의 최전선 (Multi-Agent System)

→ "AI 없으면 작동 불가"가 **허풍이 아닌 구조적 진실**이 된다.

## 8. 결론 — 수상하려면 무엇이 필요한가

### 수상 확률 현재 추정 (냉정하게)

| 심사 기준 | 현재 기획 점수 (10점 만점) | 10개 맥락 축 반영 후 |
|---|---|---|
| 활용가능성 | 7 (공공데이터 활용 우수) | 8 (+MOU 의향서 필요) |
| AI 활용도 | 6 (LLM 프롬프트 수준) | **9 (6 에이전트 파이프라인)** |
| 독창성 | 6 (핸드트래킹은 역효과) | **8 (누적 맥락 + 문화 맥락)** |
| 논리성 | 7 (Epstein 등 이론 강함) | 8 |
| 표현성 | 7 (프로토타입 전제) | 8 |
| **총점** | **33 / 50** | **41 / 50** |

→ 33점은 "장려상" 경쟁권, 41점은 "최우수~대상" 경쟁권.

### 실행 우선순위 (4주 남았을 때)

**1주차: 맥락 축 4, 6, 8 구현 (독창성 핵심)**
- 과거 공지 임베딩 DB 구축
- 출신국 문화 비교 KB (6개국 × 20토픽으로 축소)
- 가정 프로필 기반 경고 자동화

**2주차: 멀티 에이전트 파이프라인 코딩 (AI 활용도)**
- Agent 1~6 별 프롬프트 작성
- LangGraph 또는 단순 함수 체인으로 구현
- 검증 에이전트(Agent 5) 할루시네이션 탐지

**3주차: 프로토타입 완성 + A/B 테스트 50건**
- Before/After 3건 실례 슬라이드 제작
- 10개 맥락 토글 UI 구현
- MEGI 상위 3개 교육청 담당자 전화 → 의향서 확보 시도

**4주차: 기획서 재작성 + 심사위원 체험 동선**
- p.6, p.7, p.13-1 재작성
- 30초 데모 스크립트 녹화
- 발표 영상 준비
